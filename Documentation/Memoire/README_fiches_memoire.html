Fichiers Mémoire CDA – Index des livrables

Fichiers créés pour compléter le mémoire (conformité REV2/REAC). À intégrer ou copier dans le dossier de projet imprimé / PDF.

Fichier Usage
Presentation_contexte_formation.md Remplacer « Présentation de l’entreprise et du service » par le contexte formation / solo. À adapter puis insérer en section 1.3.
Maquettes_enchainement_ecrans.md Schéma Mermaid d’enchaînement des écrans (auth → jeu → replay). Exporter en image (Mermaid Live, IDE) et insérer en 3.4 + légende.
Schema_BDD_MEA_et_physique.md Modèle entité-association (MEA) et tableau du modèle physique. Insérer en 3.5 (ou annexe).
Annexe_script_BDD_auth.sql Script SQL de création + amorçage BDD auth. À mettre en annexe (script BDD).
Annexe_tableau_routes_API.md Tableau des routes API (méthode, URL, paramètres, codes). Annexe.
Annexe_extraits_code.md Extraits : step(), movement_phase_start, appel start game. Annexe.
Jeu_essai_complet.md Suggestions de jeux d’essai + exemple rédigé (démarrage PvE + mouvement). Remplir « Résultat obtenu » après exécution, puis insérer en 4.8 ou annexe.
Captures_et_code.md Structure « capture d’écran + code correspondant » pour Auth et Plateau. Ajouter tes captures puis insérer en 4.5 ou annexe.
REAC_eco_conception_et_tests.md Explication éco-conception et tests (REAC) + phrases types à insérer dans le mémoire.

Checklist : après intégration, cocher dans la checklist du mémoire : présentation contexte, maquettes, modèle EA/physique, captures+code, jeu d’essai final, annexes.
